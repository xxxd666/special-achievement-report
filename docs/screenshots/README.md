# 配图 / 截图（README 引用）

发布前，把下面这些图截好放进本目录，README 会更吸引人：

| 文件名 | 怎么来 | 用途 |
|--------|--------|------|
| `banner.png` | 浏览器打开 `docs/banner.html`，截 1280×640 的 banner 区域 | 仓库社交预览图（GitHub Settings → Social preview） |
| `01-palette.png` | 截「配色总览参考」 | 展示配色直观选择 |
| `02-report.png` | 实际生成一份汇报后截封面/KPI 页 | 展示成品效果 |

> 截图建议：浏览器全屏（F11）+ 系统截图工具；社交预览图务必 1280×640。

## 设置仓库社交预览图（强烈建议，涨 Star 关键）
GitHub 仓库 → Settings → 往下找 **Social preview** → 上传 `banner.png`。
这样别人在社交平台分享你仓库链接时，会显示这张大图，点击率显著更高。
