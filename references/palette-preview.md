# 配色总览参考（Step 4 首次用 · 生成本地 HTML 让用户直观挑色）

> 目的：用户第一次走到"选展示形态/配色"时，**与其用文字描述配色，不如直接生成一张本地 HTML 把所有可选配色平铺出来**，自动打开，让用户一眼挑。
> ⚠️ 配色是末端、是参考（原则一）——生成时就告诉用户"这只是参考，配色之后随时能单独再调"，别让它显得很重。

## 何时生成

- **首次**（用户此前没看过配色参考）：生成这份 HTML，存本地并自动打开。
- **之后**：不再每次弹窗。直接用**文字推荐 2–3 套 + 一句理由**。只有用户说"不清楚/想看看效果"时，重新打开**首次存下的那份**（路径见下），不重复生成。

## 存哪 / 文件名

- 存到当前汇报项目目录（与正在做的汇报成品同级），文件名 `配色参考-palette.html`。
- 若还没有项目目录，存到用户桌面 `配色参考-palette.html`。
- 记住这个路径，下次"想看看"时直接打开它。

## 生成规范（这份本地 HTML 长什么样）

- 单文件 HTML，内联 CSS，无外部依赖（断网也能开）。
- **三段分区，顺序固定**：① 深度汇报版（**放最上面**，横向 16:9 大样张，经典红在前、深蓝在后）→ ② 瑞士风（4 张，段标注明"来自 guizang-ppt"）→ ③ 电子杂志风（5 张，段标注明"来自 guizang-ppt"）。每段一句话说明"什么场景用"。
- **深度汇报版样张用横向 16:9**（`aspect-ratio:16/9`），与 PPT 同尺寸，并标注「16:9」；其余两段用纵向小卡。
- **每张色卡**要让人直观感受，不是只列 hex：
  - 用该主题真实底色铺卡片背景，真实字色/accent 画一行"标题大字 + 一句正文 + 一个色块/按钮 + 强调标记"的微缩样张。
  - 卡片角标注明：名称 + hex 值 + 一句"适合场景"。**命名保持中性，不写来源/公司/品牌/logo。**
- 顶部提示：**"这只是配色参考，挑个大方向就行，定稿后随时能单独调色。"**
- **底部加一行技能来源备注**：`由「专项成果汇报」技能生成 · Special Achievement Report Skill | 配色仅供参考，定稿后可单独调整`。
- 排版用 CSS grid，自适应；中文字体走系统 `PingFang SC / Microsoft YaHei` 栈。
- **不用"风格 A / 风格 B"这种内部代号**，对用户写成"电子杂志风 / 瑞士风"。
