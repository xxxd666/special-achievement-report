# 输出形态与网页 PPT 版式总览

> 本文档是「专项成果汇报」技能在 **Step 2.5 选择输出形态** 与 **Step 3 生成** 环节的衔接说明。
> 当用户选择网页 PPT(形态 ② / ③)时,本技能直接复用同目录下 `guizang-ppt-skill` 的模板与版式能力。
>
> **核心原则不变**:无论哪种输出形态,内容都来自前面 Step 1-2 用麦肯锡方法论(金字塔 / MECE / SCQA / 空雨伞)分析后的**已确认大纲**。输出形态只决定"用什么视觉容器装",不改变内容结论。

---

## 一、三种输出形态(Step 2.5 给用户三选一)

大纲确认后,**先给用户看下面这张三选一表,并给出推荐项**,等用户选定后再进 Step 3。

| 选项 | 产物 | 形态 | 底层实现 | 最适合 |
|------|------|------|---------|--------|
| **① 深度阅读版** | 单文件 HTML | **纵向滚动**,桌面深度阅读 / A4 打印 | 本技能自带规范 `html-template.md` | 向上汇报、需逐字细读、归档、打印 |
| **② 电子杂志风 PPT** | 单文件 HTML | **横向翻页**,WebGL 流体背景 + 衬线大标题 | guizang **风格 A** | 对外分享、人文 / 故事叙事、要"杂志感"的演讲 |
| **③ 瑞士国际主义风 PPT** | 单文件 HTML | **横向翻页**,极细网格 + 无衬线 + 高反差功能色 | guizang **风格 B** | 数据密集型成果汇报、KPI / 路线图 / 年度总结 |

### 自动推荐决策树(基于 Step 1 已收集的"汇报给谁看 + 风格偏好")

```
汇报给谁看?
├── 直接上级 / 需要存档 / 要打印 / 数据要逐字核对
│      └──→ 推荐 ① 深度阅读版
│           (若数据极密集且想要现代感,可加推 ③)
│
├── 对外分享 / 发布会 / demo day / 演讲台
│   ├── 内容偏人文 / 故事 / 行业观察 / 文化叙事
│   │      └──→ 推荐 ② 电子杂志风 PPT
│   └── 内容偏 KPI / 数字 / 路线图 / 工程数据 / 年度总结
│          └──→ 推荐 ③ 瑞士国际主义风 PPT
│
└── 不确定 / 既要上汇报又要对外讲
       └──→ 推荐 ①(主)+ 提示可另出 ③ 做演讲版
```

**话术示范**(给用户呈现时):
> 内容已经按金字塔结构梳理好了。接下来选一个输出形态:
> ① 深度阅读版(纵向 HTML,适合发给领导细看 / 打印)
> ② 电子杂志风 PPT(横向翻页,适合对外分享)
> ③ 瑞士国际主义风 PPT(横向翻页,数据感强,适合成果 / 年度汇报)
> 你这次是**年度成果向 VP 汇报 + 数据较多**,我推荐 **③**。要不要按 ③ 走?也可以选其他。

> ⚠️ 形态 ② 和 ③ **不能混用**(类名、模板、主题色三套都不通用),一份产物只能选一个风格。

## 二、选定网页 PPT 后:两层版式菜单

用户选了 ② 或 ③ 之后,**第一层风格已定**。第二层是:**罗列该风格的全部版式 + 主题色,并基于已确认大纲给出每页的推荐版式**,用户可逐页替换。

### 形态 ② · 电子杂志风(风格 A)—— 10 种布局

| Layout | 名称 | 成果汇报场景下怎么用 |
|--------|------|--------------------|
| **1** | 开场封面 | 汇报封面:大标题 + 时间范围 + 汇报人 |
| **2** | 章节幕封 | 每一幕("结果 / 过程 / 洞察 / 展望")的开场分隔 |
| **3** | 数据大字报 | Executive Summary 抛 3 个核心数字(满意度↑22%、成本↓15%…) |
| **4** | 左文右图(Quote + Image) | 亮点故事:左叙事右实景图(改造前后、活动现场) |
| **5** | 图片网格 | 多图实证:项目现场 / 设施改造 / 活动剪影 |
| **6** | 两列流水线(Pipeline) | 关键举措的实施流程(规划→建设→运营→优化) |
| **7** | 悬念收束 / 问题页 | 幕末过渡或结尾抛"明年如何继续" |
| **8** | 大引用页(Big Quote) | 一句话核心结论 / 领导金句 / takeaway |
| **9** | 并列对比(Before / After) | 改造前 vs 改造后、旧流程 vs 新流程 |
| **10** | 图文混排(Lead Image + Side Text) | 信息密集的图文页:主图 + 侧栏要点 |

**风格 A 主题色(5 套,只能选不能自定义)** —— 详见 `themes.md`:

| # | 主题 | 适合 |
|---|------|------|
| 1 | 🖋 墨水经典(Monocle 默认) | 通用 / 商业发布 / 拿不准就用它 |
| 2 | 🌊 靛蓝瓷 | 科技 / 研究 / 数据 / 技术发布 |
| 3 | 🌿 森林墨 | 自然 / 可持续 / 文化 / ESG |
| 4 | 🍂 牛皮纸 | 怀旧 / 人文 / 文学风 |
| 5 | 🌙 沙丘 | 艺术 / 设计 / 创意 |

### 形态 ③ · 瑞士国际主义风(风格 B)—— 22 种 S 版式

> 风格 B 进入 **Swiss locked mode**:正文页只能从 S01-S22 选,每个 `<section>` 必须写 `data-layout="Sxx"`。动手前先读 guizang 的 `swiss-layout-lock.md`,再读 `layouts-swiss.md`。

| Layout | 名称 | 成果汇报场景下怎么用 |
|--------|------|--------------------|
| **S01** | Index Cover(索引封面) | 汇报封面页 |
| **S02** | Vertical Timeline + KPI(纵向时间线+KPI) | 年度演化 / 里程碑 + 对应数据 |
| **S03** | Split Statement(分屏论点) | 一句话核心结论 / 左右分屏论点 |
| **S04** | Six Cells(六格) | 6 项工作模块定义(MECE 切分:空间 / 运维 / 服务…) |
| **S05** | Three Layers(三层) | 三层服务体系 / 治理架构 |
| **S06** | KPI Tower(KPI 塔) | **三大核心成果数字**视觉化高度差(Executive Summary 首选) |
| **S07** | H-Bar Chart(横向条形) | 5-10 项排名比较:各模块满意度 / 各项降本占比 |
| **S08** | Duo Compare(Before/After) | **改造前后 / 目标 vs 实际**对照(成果汇报高频) |
| **S09** | Dot Matrix Statement(点阵引述) | 大引述 / 年度定调 statement |
| **S10** | Split Closing(收束) | 结尾收束页 |
| **S11** | Horizontal Timeline(横向时间线) | 4-7 步实施流程 / 关键里程碑 |
| **S12** | Manifesto + Ink Banner(宣言+墨条) | 阶段性结论 / 价值主张 |
| **S13** | Three Forces(三力) | 3 个对等举措 / 三管齐下深化 |
| **S14** | Loop Form(闭环) | 服务闭环 / PDCA / 自动化流程 |
| **S15** | Matrix + Hero Stat(矩阵+总数) | 8-12 项工单 / 事项矩阵 + 一个总数据 |
| **S16** | Multi-card Brief(多卡快讯) | 6 项成果快讯小卡 |
| **S17** | System Diagram(系统图) | 智慧园区系统 / 服务生态地图 |
| **S18** | Why Now(三论点+数据) | 为什么要做这件事:3 论点 + 数据支撑 |
| **S19** | Four Cards(四卡) | 4 项等权特性 / 4 个价值维度 |
| **S20** | Stacked KPI Ledger(纵向账单) | 纵向罗列多项 KPI(成本 / 能耗 / 人效账单) |
| **S21** | Tech Spec Sheet(规格表) | 产品规格 / 系统 benchmark / 指标达成表 |
| **S22** | Image Hero(图片 Hero) | 21:9 顶图(现场 / 大屏)+ 标题 + 三列 KPI |

> **登记扩展**:`S08 + Swiss Map Component` 用于地点 / 园区分布 / 路线 / 城市关系(读 `swiss-map-component.md`,仍标 `data-layout="S08"`)。

**风格 B 版式多样性硬规则**(来自 guizang):7-8 页至少用 6 个不同 S 编号;10 页以上至少 8 个;不允许连续 3 页同一主体结构。

**风格 B 主题色(4 套,只能选不能自定义)** —— 详见 `themes-swiss.md`,**一份 deck 只用一个 accent 色**:

| # | 主题 | 适合 |
|---|------|------|
| 1 | 🔵 克莱因蓝 IKB | 科技 / 工程 / 严肃成果汇报(默认稳妥) |
| 2 | 🟡 柠檬黄 | 活力 / 创新 / 年度高光 |
| 3 | 🟢 柠檬绿 | 增长 / 效率 / 数据向好 |
| 4 | 🟠 安全橙 | 警示 / 风险 / 安全管控主题 |

## 四、生成执行:复用 guizang 的工作流

确认风格 + 每页版式后,按 guizang-ppt-skill 的 Step 2-5 执行。两个技能同在 `.claude/skills/` 下,**从本技能 `references/` 引用 guizang 的相对路径前缀为 `../../guizang-ppt-skill/`**:

| 用途 | 风格 A(电子杂志) | 风格 B(瑞士风) |
|------|------------------|----------------|
| **模板(种子文件)** | `../../guizang-ppt-skill/assets/template.html` | `../../guizang-ppt-skill/assets/template-swiss.html` |
| **版式骨架** | `../../guizang-ppt-skill/references/layouts.md` | `../../guizang-ppt-skill/references/layouts-swiss.md`(先读 `swiss-layout-lock.md`) |
| **主题色** | `../../guizang-ppt-skill/references/themes.md` | `../../guizang-ppt-skill/references/themes-swiss.md` |
| **组件手册** | `../../guizang-ppt-skill/references/components.md` | 组件细节在 `layouts-swiss.md` 附录 |
| **质量检查清单** | `../../guizang-ppt-skill/references/checklist.md` | 同左 |
| **静态校验脚本** | — | `node ../../guizang-ppt-skill/scripts/validate-swiss-deck.mjs <index.html>` |
| **地图组件(B 专用)** | — | `../../guizang-ppt-skill/references/swiss-map-component.md` |

### 执行要点(从 guizang 提炼,生成 PPT 时务必遵守)

1. **拷模板**:把对应 `template*.html` 拷到目标位置(如 `项目/XXX/ppt/index.html`),同级建 `images/` 放图片;立刻 grep `[必填]` 替换 `<title>`。
2. **选主题色**:从对应 themes 文件的 5 套(A)/ 4 套(B)里选一套,整体替换模板 `:root` 块的主题色变量。**不接受自定义 hex**。
3. **类名预检(最易踩坑)**:动手填 slide 前先 Read 模板的 `<style>` 块,确认要用的类都有定义。**风格 A / B 类名完全不通用**,同名类视觉表现也不同。
4. **主题节奏**:每页 section 带 `light` / `dark` / `hero light` / `hero dark`;不连续 3 页同主题;每 3-4 页插 1 个 hero 页。
5. **填版式**:从对应 layouts 文件复制骨架,改文案 / 数据 / 图片路径,**不要从零写**。风格 B 每页写 `data-layout="Sxx"`。
6. **数据真实**:沿用本技能铁律 —— **所有数字必须来自用户输入,绝不编造**;数据专用版式(S06/S07/S20/S21)必须填真实数据,不能用文案硬凑。
7. **自检**:风格 B 先跑 `validate-swiss-deck.mjs`;两种风格都对照 `checklist.md` 逐项过(P0 级问题必须全通过),并打开浏览器逐页做视觉核对。

> guizang 来源说明仅用于确认 skill 出处,**不要写进生成的 PPT / 封面 / 配图**。
