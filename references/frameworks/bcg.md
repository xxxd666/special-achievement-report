# BCG · 行动标题（Action Title）+ So-What 逼问

> BCG（波士顿咨询）的标志打法：**每一页的标题就是这一页的结论本身**，不是名词标签。读者光看标题连起来，就能读完整个论证。
> 在本技能里：当用户要"每页都有冲击力、领导扫一眼标题就懂"时，用它替换或叠加在麦肯锡骨架上。

## 一图看懂

```
普通做法（名词标题）          BCG 行动标题（结论句）
┌────────────────┐          ┌────────────────────────────┐
│ 能耗情况        │   →      │ 机房独立计量后能耗回落至     │
│ （然后看半天图） │          │ 2024 水平，年省 95 万       │
└────────────────┘          └────────────────────────────┘
       标题没信息                  标题自己就是结论
```

## 核心原则

| 原则 | 说明 | 常见错误 |
|------|------|---------|
| **标题即结论** | 每页标题写成完整的一句话结论，带数字 | 用"能耗分析""工单概况"这种名词标签 |
| **So-What 逼问** | 每页做完问一句"所以呢？"，答不出就说明没结论 | 摆了一堆图表却没说明它意味着什么 |
| **标题串成故事线** | 把所有页标题抽出来连读，应是一段完整论证 | 各页标题各说各的，连不成线 |
| **一页一信息** | 每页只支撑一个结论，多了就拆页 | 一页塞 3 个不相干的点 |

## 标准产物结构

不是按"章节"组织，而是按**一串结论句**组织。每页 = 一个行动标题 + 支撑它的那张图/那组数。

```
封面：一句话总结论（全篇的"所以呢"）
P1 行动标题1（结论句+数字）  + 支撑图表
P2 行动标题2（结论句+数字）  + 支撑图表
P3 行动标题3（结论句+数字）  + 支撑图表
...
把 P1~Pn 标题连读 = 完整论证
```

## 行政/园区运营实操例子

把麦肯锡式的中性标题改写成 BCG 行动标题：

| 中性标题（前） | BCG 行动标题（后） |
|---|---|
| 工单响应情况 | 抢单机制上线后响应提速 60%，满意度反超行业基准 |
| 供应商管理 | 末位淘汰 2 家拖后腿供应商，整体准时率从 87% 升到 96% |
| 能耗管理 | 机房独立计量 + 余热回收，能耗回落至 2024 水平，年省 95 万 |
| 空间改造 | 工位利用率提升至 92%，避免新增租赁支出 500 万 |

## 适用 vs 什么时候别用

| 适合 | 别用 |
|------|------|
| 给 VP/总监看的年终高光、要冲击力 | 需要完整背景铺垫、教学性汇报（标题太"满"反而跳） |
| 数据扎实、每页都有明确结论 | 数据还很毛、结论没站稳（行动标题会显得虚张声势） |
| 要做成横向 PPT、领导快速翻 | 探讨性、还在共创讨论阶段 |

> 配合麦肯锡：BCG 解决"每页标题怎么写"，骨架仍可用金字塔+MECE；分析层仍可用空雨伞出行动。
