# 金字塔之外 · BLUF / STAR / Duarte 故事弧

> 这一类收纳几种"非咨询体系"但极实用的汇报法：军方的 **BLUF**、面试/述职爱用的 **STAR**、演讲设计大师 Nancy Duarte 的 **故事弧**。各管一个特定场景。
> 在本技能里：极短汇报用 BLUF，个人述职/答辩用 STAR，要打动人的演讲/分享用 Duarte。

---

## 一、BLUF（Bottom Line Up Front · 结论先行到极致）

源自美军公文：**第一句就是结论和诉求**，后面才是支撑。比金字塔更极端——连铺垫都省。

### 核心

| 原则 | 说明 |
|------|------|
| 第一句即结论 | 开头一句话讲完"结果/请求是什么" |
| 诉求明确 | 如果要对方做决定，第一句就说要他做什么 |
| 支撑在后 | 细节、背景全部放结论之后，按需看 |

### 例子（给 VP 的极简汇报）

> **BLUF：园区全年降本 375 万、满意度 ↑22%、0 安全事故；申请 2026 预算 600 万用于二期改造。**
> （下面才是三件事怎么做到的、600 万怎么花。）

### 适合 / 别用
适合：给大忙人、IM/邮件汇报、一页纸请示。别用：需要铺陈情绪或完整论证的场合。

## 二、STAR（Situation-Task-Action-Result · 情境叙事）

讲个人贡献的黄金结构，述职、晋升答辩、个人总结必备。

### 核心

| 字母 | 含义 | 要点 |
|------|------|------|
| **S** Situation | 当时什么情况 | 简短交代背景 |
| **T** Task | 你要解决什么 | 明确你的责任/目标 |
| **A** Action | 你具体做了什么 | **重点**，突出"你"的动作和判断 |
| **R** Result | 结果如何 | 带数字，最好有对比 |

### 例子（述职里讲一个项目）

> **S** 园区设施老化、报修激增；**T** 我负责把报修响应压到 30 分钟内；**A** 我推动上线抢单系统、重谈了 2 家供应商的 SLA、设了超时预警；**R** 响应从 38 分钟降到 12 分钟，满意度反超行业基准。

### 适合 / 别用
适合：个人述职、晋升答辩、突出个人贡献。别用：团队/部门整体汇报（STAR 是"我"视角，团队汇报用麦肯锡）。

## 三、Duarte 故事弧（演讲叙事 · "what is / what could be"）

Nancy Duarte 分析顶级演讲发现：好演讲在**"现状"与"可能"之间反复拉扯**，制造张力，最后落到"新常态"。

### 核心

```
现状(what is) ↘   ↗ 可能(what could be) ↘   ↗ ...  →  新常态(号召)
              反复对比制造张力              最后给一个让人向往的结尾
```

| 要素 | 说明 |
|------|------|
| 现状 vs 可能 | 不断对比"现在这样"和"本可以那样"，拉出落差 |
| 张力驱动 | 落差就是听众想往下听的动力 |
| 情感 + 数据 | 数据立信、故事动情，两者交替 |
| 号召式结尾 | 结尾给一个清晰、向往的"新常态" |

### 例子（对外分享园区改造）
开场摆"现状"（员工挤、报修慢、体验差），抛"可能"（如果园区像家一样呢），中间用改造前后对比反复拉张力，结尾落到"我们想把每个园区都变成这样"。

### 适合 / 别用
适合：对外分享、发布会、要打动人的演讲。别用：内部数据汇报、要快速过结论的场合。

---

## 配合

- BLUF 常做**封面/开场**，后接麦肯锡主体。
- STAR 是**个人**视角，团队汇报里可用来讲"某个我主导的关键项目"。
- Duarte 故事弧管**节奏和情感**，适合形态②电子杂志风 PPT；与亚马逊 PR/FAQ 都偏叙事，可搭配。
