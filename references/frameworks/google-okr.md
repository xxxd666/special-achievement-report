# 谷歌 Google · OKR 复盘（目标 - 关键结果 - 打分 - 归因）

> OKR = Objective（目标，定性、鼓舞人）+ Key Results（关键结果，定量、可打分）。复盘时对着年初定的 O/KR **逐条打分（0~1.0）+ 归因**，是最干净的"目标达成型"汇报结构。
> 在本技能里：当用户做**季度/年度目标复盘、对着既定目标交代达成情况**时用它。

## 一图看懂

```
Objective（鼓舞人心的目标）
  ├── KR1  目标值 → 实际值 → 打分 0.7 → 为什么
  ├── KR2  目标值 → 实际值 → 打分 1.0 → 为什么
  └── KR3  目标值 → 实际值 → 打分 0.4 → 为什么（没达成的归因最重要）
        ↓
    本季 O 综合得分 + 下季调整
```

## 核心原则

| 原则 | 说明 | 常见错误 |
|------|------|---------|
| **O 定性、KR 定量** | 目标鼓舞人，关键结果可量化打分 | KR 写成没法打分的动作（"推进 XX"） |
| **逐条打分** | 每个 KR 给 0~1.0 分，客观 | 笼统说"基本完成" |
| **归因 > 分数** | 没达成的为什么、达成的靠什么，比分数本身重要 | 只报分数，不解释 |
| **0.7 是常态** | OKR 鼓励有挑战的目标，1.0 反而可能定低了 | 把没拿满分当失败 |

## 标准产物结构

```
P1 本期 OKR 总览：几个 O，整体达成率
P2-Pn 每个 Objective 一块：
   O：定性目标
   ├ KR1：目标→实际→分→归因
   ├ KR2：目标→实际→分→归因
   └ 小结：这个 O 做成/没做成的根因
P尾 下一期：基于复盘，O/KR 怎么调
```

## 行政/园区运营实操例子

```
O：让园区成为"员工愿意来、来了不想走"的办公空间
├ KR1 服务满意度 ≥4.5 → 实际 4.2 → 打分 0.6
│   归因：响应速度上来了，但餐厅排队拖了后腿
├ KR2 人均办公成本 ↓15% → 实际 ↓15% → 打分 1.0
│   归因：供应商重招 + 能耗优化双管齐下，超预期
└ KR3 安全事故 0 起 → 实际 0 起 → 打分 1.0
本季综合 0.87；下季把"餐厅体验"单独拎成一个 KR 主攻
```

## 适用 vs 什么时候别用

| 适合 | 别用 |
|------|------|
| 团队/个人对着年初目标做复盘 | 一次性专项、本来就没定 OKR 的工作 |
| 目标管理文化的公司 | 探索性、目标本身还在变的事 |
| 季度/年度节点同步达成情况 | 对外讲故事、争取认可（OKR 偏内部交代） |

> 配合：OKR 提供"对目标交代"的骨架；归因部分可借空雨伞（事实→解读→行动）深化；视觉上每个 KR 的"目标 vs 实际"用进度条/打分条（瑞士风 S07 H-Bar）最直观。
