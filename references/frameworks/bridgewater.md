# 桥水 / 名人 · 信号 - 诊断 - 原则 - 行动（达里奥式原则提炼）

> 桥水创始人瑞·达里奥（Ray Dalio）的方法核心：遇到事不止解决它，而是**把它抽象成一条可复用的"原则"**，下次同类问题直接套用。汇报上体现为：从具体现象提炼出经验原则，让复盘有沉淀价值。
> 在本技能里：当用户做**经验沉淀 / 复盘会 / "踩过的坑和学到的东西"**类汇报时用它——它让"我们做了什么"升级成"我们悟到了什么、以后怎么办"。

## 一图看懂

```
信号（具体发生的事）
   ↓ 别只就事论事
诊断（根因是什么，背后的规律）
   ↓ 抽象一层
原则（可复用的一句话经验：以后遇到 X 就 Y）
   ↓ 落地
行动（把原则固化成 SOP/机制/清单）
```

## 核心原则

| 原则 | 说明 | 常见错误 |
|------|------|---------|
| **痛苦 + 反思 = 进步** | 把问题/失败当提炼原则的素材 | 复盘只检讨，不沉淀 |
| **抽象成原则** | 从一件事提炼出可复用的规律 | 停在"这次没做好下次注意" |
| **原则可复用** | 原则应能套用到未来同类情况 | 原则太具体，换个场景就不适用 |
| **固化成机制** | 好原则要变成 SOP/清单/制度 | 原则只停在嘴上和 PPT 里 |

## 标准产物结构

```
P1 本期关键事件/信号（做成了什么、踩了什么坑）
P2-Pn 每条沉淀一块：
   信号：发生了什么
   诊断：根因/规律
   原则：提炼出的一句话经验
   行动：固化成什么机制/SOP
P尾 原则清单：把本期所有原则汇成一份"团队作战手册"
```

## 行政/园区运营实操例子

| 信号 | 诊断 | 原则 | 行动 |
|------|------|------|------|
| 大型员工活动参与率从 35% 跌到 28% | 大而全的活动反而无人响应，小兴趣组却火 | **"社群 > 大活动"：员工要的是归属感，不是热闹** | 2026 活动预算转向兴趣社群运营，设社群运营岗 |
| 设施报修激增，事后才发现是维护没跟上 | 重救火、轻预防 | **"预防性维护省的钱，远超救火花的钱"** | 把预防性维护完成率纳入供应商 SLA 硬考核 |
| 跨部门需求老是扯皮 | 行政服务没有 SLA 承诺，权责不清 | **"没有 SLA 的服务等于没有承诺"** | 出行政服务目录 + SLA，关键需求 2h 响应 |

## 适用 vs 什么时候别用

| 适合 | 别用 |
|------|------|
| 复盘会、经验沉淀、"教训与心得" | 报数据成果、争资源（它偏内省不偏展示） |
| 想让汇报有"可复用价值"而非一次性 | 极短汇报（提炼原则需要思考篇幅） |
| 团队学习型、重视方法论沉淀的氛围 | 纯对外宣传 |

> 配合：信号-诊断 部分可借麦肯锡议题树（Why Tree）做根因分析；原则-行动 与空雨伞的"伞"一脉相承。适合做复盘类汇报的"经验沉淀"章节。
