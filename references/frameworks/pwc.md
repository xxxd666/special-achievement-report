# 普华永道 PwC（四大）· 审计式 Findings：事实-标准-差距-影响-建议

> 四大（PwC/德勤/EY/毕马威）做审计和风险报告时的标准句式：每一个发现（Finding）都拆成五段，**客观、不带情绪、可追溯、直指整改**。
> 在本技能里：当用户要做**复盘 / 自查 / 风险盘点 / 问题清单**时用它——它最擅长"把问题讲清楚还不显得在甩锅"。

## 一图看懂

```
一个 Finding = 五段式
┌─────────┬──────────────────────────────┐
│ 事实     │ 客观发生了什么（带数据）       │
│ 标准     │ 本应达到的基准/目标/规范       │
│ 差距     │ 事实与标准差多少               │
│ 影响     │ 这个差距对业务意味着什么风险   │
│ 建议     │ 怎么整改、谁来做、何时        │
└─────────┴──────────────────────────────┘
```

## 核心原则

| 原则 | 说明 | 常见错误 |
|------|------|---------|
| **事实先行、客观** | 先摆可核查的事实，不带评价词 | 一上来就下判断"做得差" |
| **对标准** | 每个事实都对一个明确基准 | 只说现状，不说"本该多少" |
| **影响量化** | 把差距翻译成业务风险/损失 | 列了问题但说不清危害 |
| **建议可执行** | 给到"做什么+谁+何时" | 建议停在"应加强管理"层面 |

## 标准产物结构

```
P1 概览：本次自查范围 + 发现总数 + 按严重度分级（高/中/低）
P2-Pn 每个 Finding 一块（或一行表）：事实 | 标准 | 差距 | 影响 | 建议
P尾 整改计划：按优先级排期，责任人 + 时间节点
```

每个 Finding 用一行表格呈现最清晰：

| # | 事实 | 标准 | 差距 | 影响 | 建议 |
|---|------|------|------|------|------|
| 1 | … | … | … | … | … |

## 行政/园区运营实操例子

| 事实 | 标准 | 差距 | 影响 | 建议 |
|------|------|------|------|------|
| 2025 预防性维护仅完成计划的 42% | 行业惯例 ≥90% | 缺口 48pct | 设施故障率上升、报修量 ↑120% | Q1 补做积压维护，纳入供应商 SLA 考核，月度跟踪 |
| 消防通道 3 处被占用 | 消防规范 0 占用 | 3 处 | 验收不通过 + 安全责任风险 | 本周清理 + 张贴禁占标识 + 纳入日巡 |
| 2 家核心供应商准时率 < 80% | 合同约定 ≥95% | 15pct+ | 拖累整体履约、影响业务交付 | 季度述职 + 末位淘汰机制，1 家限期整改 |

## 适用 vs 什么时候别用

| 适合 | 别用 |
|------|------|
| 专项自查、风险盘点、问题复盘 | 报喜的成果高光汇报（审计味太重像在挑刺） |
| 需要客观、可追溯、能落到整改 | 对外宣传、讲故事打动人的场合 |
| 给上级/审计/合规看 | 想突出团队功劳和正向价值时 |

> 配合：PwC findings 适合做麦肯锡汇报里的"风险/问题"那一页（或那一节）；出了 Findings 再用空雨伞把"建议"延展成行动计划。
